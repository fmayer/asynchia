 19  """ A domain-specific language to conveniently define the structure of packets 
 20  in binary protocols. 
 21   
 22  An elementary expression is an object implementing the Expr interface (which 
 23  is preferably an interface of a subclass of Expr and thus consistenly 
 24  integrates with other Exprs). It needs to implement the __call__(state) 
 25  and the produce(value) methods where __call__ returns the appropriate 
 26  collector to collect the data described by the expression and produce 
 27  returns the input that encodes the value passed to it according to the 
 28  expression. 
 29   
 30  By default the following expressions are defined by the module:: 
 31   
 32      - BinaryExpr (BE) 
 33      - SingleBinaryExpr (SBE) 
 34      - StringExpr (SE) 
 35      - FixedLenExpr (FLE) 
 36      - FileExpression (FE) 
 37   
 38  Two methods are provided to lookback data earlier collected in order to 
 39  construct the collector. binarylookback(ind, item=0) (also exported as bl) 
 40  gets the item specified from the binary data parsed from the expression at 
 41  the index ind (zero-index) of the ExprAdd the respective expression is 
 42  contained within. It is also possible to use positions relative to the 
 43  current one (-1 is the element before the one the lookback is passed to). 
 44  singlebinarylookback (also exported as sbl) does likewise for 
 45  SingleBinaryExpr (and is arguably more useful, see the Example section). 
 46  The only Expr in this module that takes a lookback as a parameter is 
 47  FixedLenExpr which limits whichever expression is contained in it 
 48  to the number of bytes aquired by the lookback function. 
 49   
 50  Attributes can be named by passing the name in square brackets. It is 
 51  also possible to refer to expressions by the name given to them in 
 52  lookbacks. :: 
 53   
 54      e = s.L()['size'] + s.B()['blub'] + LFLSE('size')['string'] 
 55   
 56  The tuple of values collected can be converted into a dictionary that 
 57  maps the name of the expression to the respective value by calling the 
 58  tonamed method of the ExprAdd. :: 
 59   
 60      a = e() 
 61      [Collecting of data takes place] 
 62      tup = a.values 
 63      dic = e.tonamed(tup) 
 64   
 65  Example 
 66  ======= 
 67  Packets are described by expressions which can be converted to collectors 
 68  whenever a packet of that type should be parsed. Complex expressions can 
 69  be created by adding expressions, resulting in ExprAdds (but that is semi- 
 70  internal API which you do not necessarily need to care about, unless you 
 71  try to write your own types of expressions). 
 72   
 73  Let us henceforth consider the following simple example:: 
 74       
 75      from asynchia.dsl import s, SBLFLSE 
 76      e = s.L() + s.B() + LFLSE(0) 
 77   
 78  This might appear utterly complicated at first glance, but it is not. The 
 79  first statement imports s (which is a container for binary numeric types) 
 80  and LFLSE which expands to lookback fixed-length string-expression. 
 81   
 82  The expression (which is the second statement) describes a packet which 
 83  contains three parts. The first part is an unsigned long (which is named by its 
 84  name in the struct module) in network byte-order (all binary types contained 
 85  in `s` assume network byte-order (so s.L expands to the struct format "!L"); 
 86  the second part is an unsigned byte; the interesting thing in the expression 
 87  is the third part which describes a fixed-length string with the length equal 
 88  to the first element in the expression (which is referred to by its index 0): 
 89  the unsigned long. 
 90   
 91  The expression can now be used to parse and to produce packets according to the 
 92  given format. An asynchia.ee collector can be created by calling the 
 93  expression :: 
 94   
 95      collector = e() 
 96   
 97  You can now pass the protocol to the add_data method of the collector and it 
 98  will collect the data as specified. Any other data (which appears after the 
 99  whole packet has been read) is left in the protocol. 
100   
101  To create a packet you need to call the expression's produce method and pass 
102  a tuple of the data you want to construct a package of. This operation 
103  returns an asynchia.ee.Input that can directly be passed to be sent by 
104  an asynchia.Handler. :: 
105   
106      data = e.produce((5, 2, 'ABCDE')) 
107   
108  In this case it is important that the first number equals the length of the 
109  string, as the system does not derive it from the length of the string 
110  (though you are advised to do so in your client code). 
111  """
