Class Hierarchy

  • object: The most base type
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • asynchia.ee.Depleted
    • asynchia.ee.Collector: This is the base-class for all collectors.
      • asynchia.ee.CollectorQueue: Write data to the first collector until CollectorFull is raised, afterwards repeat with next.
        • asynchia.ee.KeepingCollectorQueue: CollectorQueue that does not discard the collectors contained within after they have finished but rather appends to them to the list accessible through its member collected.
      • asynchia.ee.DelimitedCollector: Collect up to size bytes in collector and raise CollectorFull afterwards.
        • asynchia.ee.StructCollector: Collect and unpack stru.
          • asynchia.ee.SingleStructValueCollector
      • asynchia.dsl.ExprCollectorQueue: Queue to collect the data specified by the list of expressions passed to it.
      • asynchia.ee.FactoryCollector: Call factory method to obtain the next collector until the factory raises the Depleted exception.
      • asynchia.ee.FileCollector: Write data received from the socket into a fd.
      • asynchia.ee.StringCollector: Store data received from the socket in a string.
    • asynchia.dsl.Container: Container to hold members, which are also exposed through getitem.
    • asynchia.forthcoming.Coroutine: Create coroutine from given iterator.
    • asynchia.forthcoming.DataNotifier: Call registered callbacks and send data to registered coroutines at submission of data.
    • asynchia.dsl.Expr: Baseclass for all expressions.
      • asynchia.dsl.BinaryExpr: Represents data that can be expressed by a Struct.
      • asynchia.dsl.ExprAdd
      • asynchia.dsl.FileExpr: Data that should be collected in an fd.
      • asynchia.dsl.FixedLenExpr: Restrict the length of the expression contained to the value retuned by calling glen upon construction of the collector for said expression.
      • asynchia.dsl.SingleBinaryExpr: Represents data that can be expressed by a Struct with a single member.
      • asynchia.dsl.StringExpr: Data that should be collected in a string.
    • asynchia.util.GradualAverage: Memory-efficient average to which values may gradually be added over time.
    • asynchia.Handler: Handle a socket object.
      • asynchia.AcceptHandler: Handle socket that accepts connections.
        • asynchia.Server: Automatically create an instance of handlercls for every connection.
      • asynchia.ee.Handler: asynchia handler that adds all received data to a collector, and reads outgoing data from an Input.
      • asynchia.protocols.LineHandler: Use this for line-based protocols.
      • asynchia.forthcoming._ThreadedDataHandler: Implementation detail.
    • asynchia.util.IDPool: Pool that returns unique identifiers.
    • asynchia.ee.Input: Base-class for all Inputs.
      • asynchia.ee.FactoryInput: Call factory method to obtain the next input until the factory raises the Depleted exception.
      • asynchia.ee.FileInput: Input that buffers at most buffer_size bytes read from the passed fd, and sends them in a buffered way.
        • asynchia.ee.AutoFileInput: Same as FileInput, only that it stores samples to guess how big the buffer should be.
      • asynchia.ee.InputQueue: An InputQueue calls the tick method an object until it raises InputEOF, then it goes on with the next object in the queue.
      • asynchia.ee.StringInput: Input that bufferedly sends a string.
    • asynchia.maps.InterruptContextManager: Allow with socketmap.interrupt()
    • asynchia.util.LimitedAverage: Average considering the last samples values added to it.
    • asynchia.ee.MockHandler: This mocks a handler by writing everything that's passed to its send method to outbuf, while reading data from inbuf upon recv calls.
    • asynchia.Notifier: Call handle functions of the object with error handling.
    • asynchia.forthcoming.PauseContext: Collection of Coroutines which are currently paused but not waiting for any data.
    • asynchia.SendallTrait: Enhance Transport with sendall that buffers data and sends it all, if necessary in multiple steps.
    • asynchia.SocketMap: Decide which sockets have I/O to be done and tell the notifier to call the appropriate methods of their Handle objects.
      • asynchia.maps.ControlSocketSocketMap: Socket-map with an internal socket-pair that can be used to interrupt it.
        • asynchia.maps.FragileSocketMap: The socket-map has to be interrupted before internals are changed.
          • asynchia.maps.SelectSocketMap: Decide which sockets have I/O to do using select.select.
        • asynchia.maps.RobustSocketMap: The socket-map has to be stopped and resumed after internals are changed.
          • asynchia.maps.PollSocketMap: Decide which sockets have I/O to do using select.poll.
        • asynchia.maps.RockSolidSocketMap: The socket-map automatically refreshes its internal while it's "sleeping".
          • asynchia.maps.EPollSocketMap: Decide which sockets have I/O to do using select.epoll.
      • asynchia.qtmap.QSocketMap: Decide which sockets have I/O to do using Qt facilities.
    • asynchia.dsl.State: State of the collection done by ExprCollectorQueue.
    • asynchia.Transport: All I/O is done over Transports.
      • asynchia.SocketTransport: A transport that uses a socket to send and receive data.
    • int: int(x[, base]) -> integer
      • PyQt4.QtCore.QSocketNotifier.Type
    • sip.simplewrapper
      • sip.wrapper
        • PyQt4.QtCore.QObject: QObject(QObject parent=None)
          • PyQt4.QtCore.QSocketNotifier: QSocketNotifier(int, QSocketNotifier.Type, QObject parent=None)
            • asynchia.qtmap.SocketNotifier: Notificate about socket I/O using Qt facilities
  • asynchia.forthcoming.Coroutine.return_: Yield an instance of this to signal that the coroutine finished with the given return value (defaults to None).
